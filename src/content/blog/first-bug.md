## 问题描述

今天在搞一个 API 接口，死活调不通。报错信息就一个：`401 Unauthorized`。

我寻思着这不就是认证失败嘛，检查 token、检查权限、检查接口地址，全都没问题。

然后就开始怀疑人生了。

## 排查过程

### 第一步：检查 token

token 是从登录接口拿的，fresh 的，没过期。

```javascript
const token = localStorage.getItem('token');
console.log(token); // 有值，看起来也正常
```

### 第二步：检查请求头

请求头也没问题，格式对的：

```javascript
headers: {
  'Authorization': `Bearer ${token}`
}
```

### 第三步：检查接口地址

地址也没错，测试环境、生产环境都试了，都报同样的错。

### 第四步：开始怀疑人生

问了同事，同事说你这看着没问题啊。

我说对啊，就是没问题才诡异。

## 真相大白

折腾了两个多小时，最后你猜怎么着？

**后端改接口了，没跟我说。**

新接口要加个 `tenantId` 参数，我没加，所以一直 401。

问后端为啥不早说，他说："我以为你知道啊。"

我说："我要是知道还能在这浪费两小时？"

## 解决方案

加个参数就行：

```javascript
const params = {
  tenantId: 'xxx',
  // 其他参数...
};
```

完事儿。

## 教训

1. **接口变动一定要同步** - 尤其是这种加必填参数的
2. **错误信息要友好** - `401` 谁看得出来是缺参数
3. **沟通很重要** - 别以为别人知道你在想啥

## 最后

这个 Bug 虽然弱智，但确实花了我一下午。记下来，提醒自己以后别犯同样的错。

也提醒各位后端的同学：**接口改了，记得说一声 🙏**

就这样，回见。
